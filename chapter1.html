Chapter 1 of 19

Data as a Matrix

A bank wants to know: will this loan applicant default? They have records of past applicants — income, age, debt ratio — and whether each person repaid or defaulted. Before any ML algorithm can run, this raw table must become a matrix. This chapter shows exactly how and why.

Our dataset — loan applicants

We have 5 past loan applicants. For each person we know their income ($k), age, and whether they defaulted (1 = defaulted, 0 = repaid). This is our training data.

Raw data table

# (applicant) Income ($k) Age Default (0/1)

■ Blue = features (inputs to the model)  |  ■ Gold = target (1 = defaulted, 0 = repaid)

Why a table? In ML, every data point (loan applicant) is one row. Every property we measure (income, age) is one column. This uniform structure is what lets us do matrix math on it.

Step 1 — Build the design matrix X

We take the feature columns (income, age) and prepend a column of 1s called the bias column. This lets our model learn an intercept.

Why add a column of 1s?

Our prediction formula is: score = θ₀·1 + θ₁·income + θ₂·age
The θ₀ term is a constant — it has no feature multiplying it. By adding a column of 1s, every term looks the same: feature × weight. This makes the matrix equation clean.

score = θ₀·1 + θ₁·income + θ₂·age   →   [1, income, age] · [θ₀, θ₁, θ₂]ᵀ
Bias (always 1)
Features (income, age)

5 rows = 5 applicants  ·  3 columns = bias + 2 features (income, age)

Step 2 — Extract the target vector y

The default column becomes a separate vector y. Each entry is 1 (defaulted) or 0 (repaid) — this is what we're trying to predict.

y — Target vector (5 × 1)

[
]

5 rows = 5 default labels (1 = defaulted, 0 = repaid)

Step 3 — The full picture: [X | y]

Combining them gives the augmented matrix — the same form used in row reduction. The goal of ML is to find θ such that Xθ ≈ y.

[X | y] — Augmented matrix (5 × 4)

[
]
The ML problem in one line: Find vector θ such that Xθ ≈ y. Every ML algorithm is a different strategy for solving this.

Matrix shapes — why they matter

X shape: 5 × 3
  • 5 rows = 5 training examples
  • 3 columns = 3 parameters (bias + 2 features)
y shape: 5 × 1
  • 5 rows = one label per applicant
  • 1 column = single output (0 or 1)
θ shape: 3 × 1 (what we want to find)

θ = [θ₀, θ₁, θ₂]ᵀ — three weights, one per column of X. For Xθ to be valid: (5×3) × (3×1) = (5×1) ✓

Scaling to real ML: Real datasets have millions of rows (data points) and hundreds of columns (features). The matrix structure is exactly the same — just larger. The math doesn't change.
Story so far: We have 5 loan applicants as feature matrix X (5×3: bias, income, age) and target vector y (5×1: default labels).
