Gradient Descent

Ch 4 found the optimal θ by inverting a matrix — elegant, but O(n³). For millions of applicants or thousands of features, that's too slow. And if features are correlated (Ch 5), the matrix may not even be invertible. Gradient Descent finds the same θ iteratively — no inversion needed.

Why not always use the Normal Equation?

Normal Equation Gradient Descent
How it works Exact formula θ=(XᵀX)⁻¹Xᵀy Iterative updates
Complexity O(n³) — cube of features O(k·m·n) — linear in data size
For 1000 features 10⁹ operations — slow Fast, works well
Requires tuning No (one-shot) Yes (learning rate α)

The intuition — a ball rolling downhill

Imagine the loss L(θ) = ‖Xθ − y‖² as a bowl-shaped surface. The minimum of the bowl is the optimal θ. Gradient Descent:

  1. Start at a random point on the bowl
  2. Compute the gradient — which direction is steepest uphill?
  3. Step a little in the opposite direction (downhill)
  4. Repeat until the ball settles at the bottom
θ ← θ − α · ∇L(θ)   where   ∇L = (2/m) Xᵀ(Xθ − y)

Loss surface — 2D contour (θ₀ vs θ₁)

Colour = loss value (dark = low, bright = high). The white dot is the current θ. Watch it descend to the minimum (red ★).

The gradient in matrix form: ∇L = (2/m) Xᵀ(Xθ − y) This is a single matrix–vector product — very fast to compute even for large datasets. That's why gradient descent scales to millions of examples while the Normal Equation doesn't.
Learning rate matters: Too large → overshoots, diverges. Too small → converges slowly. Try different values of α using the slider above to see both effects.
