Solving with the Normal Equation

In Ch 3 we saw that ŷ = Xθ — but we were guessing θ. Now we derive the formula that finds the optimal θ in one algebraic step: θ = (XᵀX)⁻¹Xᵀy. This is row reduction applied to our loan applicant data.

The formula:   θ = (XᵀX)⁻¹ Xᵀy   — Click each step below to build intuition for where this comes from.
Step 1 Why can't we just solve Xθ = y directly? ▼

Our system is overdetermined: 5 equations (applicants) but only 3 unknowns (θ₀, θ₁, θ₂). No exact solution exists — there's no θ that makes Xθ = y exactly.

5 equations, 3 unknowns. The system is overdetermined — we can't satisfy all 5 equations exactly. Instead, we find θ that minimises the error: min‖Xθ − y‖².
Step 2 Compute Xᵀ and XᵀX ▼

Multiplying both sides of Xθ = y by Xᵀ gives a square system: XᵀXθ = Xᵀy. Now we have 3 equations and 3 unknowns — solvable!

Why does this work? Multiplying by Xᵀ projects the overdetermined problem into a space where a unique solution exists. The resulting XᵀX is always square (3×3) regardless of how many data points you have.

Step 4 Read off θ — the best-fit parameters ▼

The RREF gives us θ directly in the last column. These are the same weights you explored interactively in Chapter 2 and 3.

This is the optimal solution. No other θ produces a smaller sum of squared errors for this dataset. The Normal Equation finds it in a single computation.
Step 5 When does it fail? Rank & singularity ▼

The Normal Equation requires XᵀX to be invertible (full rank). Two common situations where it breaks:

⚠ Problem 1: Linearly dependent features

If "age" = "income × 0.6" exactly (perfectly correlated), the columns of X are linearly dependent. XᵀX becomes singular — determinant = 0 — no unique inverse exists.

In ML terms: Redundant / perfectly correlated features → model can't distinguish which one matters → infinite solutions.

⚠ Problem 2: More features than data points

If n (features) ≥ m (data points), rank(X) < n → XᵀX is singular. This is common in modern ML (e.g. 1000 features, 50 examples).

Fix: Add regularisation (Ridge regression adds λI to XᵀX) or switch to gradient descent.

This is why rank matters. rank(XᵀX) = rank(X). If X doesn't have full column rank, the Normal Equation has no unique solution — your features contain redundant information.

The complete ML pipeline — in matrix form

Represent data Feature engineering X ∈ ℝ^(m×n)
Make predictions Forward pass ŷ = Xθ
Measure error Loss function ‖Xθ − y‖²
Find best θ Training θ = (XᵀX)⁻¹Xᵀy
New prediction Inference ŷ_new = x_new · θ
You've now solved the loan model end-to-end using the Normal Equation. But wait — what happens if income and age are highly correlated? That makes XᵀX singular and the Normal Equation fails. Chapter 5 investigates exactly this.
