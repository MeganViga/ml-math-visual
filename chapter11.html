From Scores to Probabilities

Linear regression (Ch 1–7) gave us a number: predicted house price. For loan default, we need a probability between 0 and 1. The fix is one function: the sigmoid σ(z) = 1 / (1 + e⁻ᶻ). This is the bridge from linear algebra to probability.

The logistic regression pipeline

Features x
[income, age, debt]
→
Linear score
z = xᵀθ
→
Sigmoid
σ(z) = 1/(1+e⁻ᶻ)
→
P(default)
ŷ ∈ [0,1]

Step 1 is exactly Ch 3's matrix multiplication z = Xθ. Step 2 squashes any real number to (0,1).

Why this matters for the loan model: The sigmoid ensures our output is always a valid probability. When P(default) > 0.5, we decline the loan. The threshold can be adjusted — Chapter 17 will show how changing the threshold affects false positives vs false negatives.
Story so far: We've added the sigmoid on top of the linear model from Ch 1–7. Now we output P(default). But how do we find the best θ for a probability output? We can't use the Normal Equation — logistic regression has no closed-form solution.
