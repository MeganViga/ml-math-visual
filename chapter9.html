Which Features Predict Default?

In Ch 8 we looked at each feature separately. Now we ask: which features move together with the default outcome? That relationship is called correlation — and it tells us which inputs will be most useful to our model.

Feature vs Default — scatter plots

Each dot is one applicant. Red = defaulted, green = repaid. Look for clear separation — that's a predictive feature.

Debt Ratio vs Default

Pearson correlation with default outcome

Correlation r ∈ [−1, +1]. A negative r means higher feature → less likely to default. |r| near 1 = strong predictor. Near 0 = weak predictor.

Feature r with Default Strength Direction

Feature–feature correlation matrix

This is the covariance matrix idea from Ch 7 — applied to our loan features. High correlation between features (like income & loan amount) is the multicollinearity that caused rank issues in Ch 5.

Correlation heatmap — blue = positive, red = negative

Features: Income, Age, Loan, Debt ratio, Default (y)

Why this matters for the loan model: Income and age are negatively correlated with default — higher income & older applicants default less. Debt ratio is positively correlated. These correlations guide feature selection, and we also see that income and loan amount are correlated with each other — potential multicollinearity to watch for (Ch 5 lesson applied here).
Story so far: Income, age, and debt ratio all correlate with default outcome. The features also correlate with each other. This confirms they're all useful — but we can't just read off who will default.
