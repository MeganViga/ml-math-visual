Partial Derivatives & The Gradient

Ch 14 showed the derivative of L with respect to a single θ. But our loan model has θ₀ (bias) and θ₁ (income weight) — loss depends on both simultaneously. The gradient ∇L is the vector of all partial derivatives: it points uphill in parameter space. Gradient descent follows −∇L.

The partial derivatives for logistic regression

For each parameter θⱼ, the partial derivative is the same elegant formula from Ch 14:

∂L/∂θ₀ = (1/m) Σ (σ(z_i) − y_i)  bias gradient
∂L/∂θ₁ = (1/m) Σ (σ(z_i) − y_i) · income_i  income weight gradient

Together: ∇L = [∂L/∂θ₀, ∂L/∂θ₁] — a 2D vector pointing uphill. Gradient descent takes steps in the direction of −∇L.

Each contour ring is a constant loss value. Click any point (θ₀, θ₁) to see the gradient arrow pointing uphill. The minimum (deepest valley) is where gradient descent ends up.

Why this matters for the loan model: The gradient tells us simultaneously how to adjust the bias and the income weight. In our full loan model with 4 features (income, age, loan, debt), we'd have 5 partial derivatives — one per θ — all computed and applied in parallel each step.
